# 🚀 Novelove システム引継ぎ書 (v14.0.0 ベース)

> **作成日**: 2026-04-09
> **概要**: 専売タグ問題の完全解決から、ファイル分割リファクタリングによるv14到達までのシステム現状サマリー。次回以降の開発・保守を行うためのコンテキストを網羅しています。

---

## 1. 最近の重大なバグ修正 (v13.7.x - 13.9.x)

### 1-1. 専売・独占タグの完全復旧
v13.5.1で発生した「タグ名変更による全消失」事故から始まり、サイトごとの判定精度を100%に引き上げました。
- **DigiKet**: 専売判定 `scrape_digiket_description()` の6番目の戻り値（`is_exclusive`）の受け漏れという致命的バグを修正。既存DBの全件再スクレイピング（バックフィル）を行い、正常化。
- **FANZA / DLsite / らぶカル**: WordPressのタグとローカルデータベースの整合性を修復。
- （※ランキング生成時に発生した、DigiKetの戻り値unpackエラーも **v13.7.3** で解消済み）

### 1-2. セキュリティとパフォーマンスの向上 (v13.9.0)
- **セキュリティ**: FIFU（アイキャッチ）設定時に `wp eval` を用いていた箇所を `wp post meta update` へ置換。外部の画像URLを介したPHPコード注入リスクを完全排除。
- **パフォーマンス**: タイトル重複判定 (`is_cross_db_duplicate()`) で全件の類似度計算を行うO(N)問題を解消。`LIMIT 1000` および文字数差による事前足切りを導入し、数千件規模のDBでのCPUスパイクを回避。

## 2. アーキテクチャの刷新 (v14.0.0)

ファイル肥大化の限界を迎えつつあった `auto_post.py`（1816行）を3つに分割し、責務を明確化しました。今後の機能追加やプロンプト修正は、対象のファイルのみを編集すれば済みます。

### 📁 依存関係 (一方向化)
`soul` → `core` → `fetcher` → `writer` → `ranking` → `auto_post`

### 📄 各ファイルの責務
1. **`novelove_writer.py`** (執筆エンジン)
   - AIへのプロンプト構築 (`build_prompt`)、DeepSeek API通信、記事生成、スコア/おすすめ要素の動的注入。
   - **戻り値**: v13.10.0 で導入された **`ArticleResult` dataclass** を返し、タプルや配列インデックスへの直アクセスを廃止。

2. **`novelove_ranking.py`** (ランキング生成)
   - クロール（DMM/FANZA, DLsite, DigiKet）、ランキング記事用プロンプト構築、ランキングとしてのWP投稿処理。

3. **`auto_post.py`** (メインコントローラー)
   - スレッド制御、WPへの通常記事投稿 (`post_to_wordpress`)、緊急停止/重複チェック監視などの「司令塔」機能のみを保持。(764行にスリム化)

## 3. 次回開発者・運用者への申し送り事項

1. **スクレイピングと判定ロジックの変更時**
   `novelove_fetcher.py` などでタプルの戻り値を増減させた場合、`auto_post.py` や `novelove_ranking.py` などの全呼び出し元で確実に個数をあわせること（DigiKetの受け漏れバグの教訓）。可能であれば、`ArticleResult` のような `dataclass` や NamedTuple への移行を徐々に進めることが推奨されます。

2. **タグ名・カテゴリ名の変更絶対禁止**
   「FANZA独占」等のWordPress上のタグ名は、サイトフィルターと強く結合しています。名称を変更すると過去数千件の記事のタグが迷子になるため、変更が必要な場合は必ず `SPECIFICATIONS.md` を更新し、DB更新スクリプトを連動させること。

3. **ロックファイルについて**
   もし自動投稿が完全に停止した場合、`nexus_rewrite.py` や `auto_post.py` が生成した `.lock` ファイルが残存している可能性があります。現在はタイムアウトによる「Staleロック自動解除機能（1時間経過後）」が実装されていますが、即時復旧が必要な場合は手動で削除してください。

4. **SEO・GSCのAPI制限**
   `nexus_gsc.py` で使用する Google Search Console API の1日あたりのインデックス登録リクエスト上限は公式の `600` に設定されています（v13.8.2）。これを引き上げないよう注意してください。
