# 【インシデント報告】DMM APIの「独占」ラベルノイズによる専売タグの過剰付与問題

**日付**: 2026-04-13
**対象**: Noveloveシステム (Fetcher, Writer, Rewrite Engine, DB)

## 1. 発生した問題
- `d_751438` などの「らぶカル」の作品において、本来「専売」ではないにもかかわらず、システムが誤って `FANZA独占` などの専売タグを付与し、さらにデータベース上で `is_exclusive = 1` として記録してしまう不具合が発生した。
- これによって、サイドバーの「専売・限定作品」ナビゲーション等にも意図しない形で多数の作品が表示される事態となった。

## 2. 根本原因 (Root Cause)
現在のNoveloveシステムの「専売・限定判定」は、以下の2段構えで実行されている（`novelove_fetcher.py` L878〜）。
1. **DOMスクレイピング**: 商品ページのHTMLに `.c_icon_exclusive` クラスのバッジが含まれているかを直接確認する。（**正確**）
2. **APIジャンルデータ検索**: DMMのAPIが返す `iteminfo.genre` や `iteminfo.label` の中に「独占」「専売」という文字列が含まれているかを確認する。（**ノイズが含まれる**）

今回、DMMの公式APIが**「らぶカル」の作品群自体に対して、商品ページに専売バッジが存在しなくてもAPI裏側で『【独占】らぶカル』などのラベルを付与して返却してくる**ことが判明した。
システムは②の「APIデータの検索」でこれらのノイズを拾い上げ、誤って `is_exclusive = 1` をTrueとして記録してしまっていた。

## 3. 現在の暫定対応
1. **フェッチャー・リライトエンジンのロジック修正パス**: ユーザーと協議の結果、現状のコードによる「APIデータを信じるか否か」の修正方針（らぶカル一律除外など）は保留中。
2. **サイドバー非表示処置**: ユーザーからの「まずは一旦表示を消して後日調査する」という指示を受け、WordPressの `functions.php` において、`[novelove_navigator_v3]` が出力している **「専売・限定作品」のタググループ（HTML ul要素）を全体的にHTMLコメントアウトして非表示化** した。
   - `functions.php` の `<!-- TEMPORARY HIDDEN BY AI: EXCLUSIVE TAGS ... -->` の部分が該当。

## 4. 今後の恒久対応（Next Steps）
後日、時間がある時に以下の対応を実施する必要がある。

1. **フェッチャーのロジック修正**:
   - `novelove_fetcher.py` の `is_excl_api` （APIから「独占」という文字を拾うロジック）を完全に削除するか、あるいはFANZA/DMMブックス等では信用して良いのかを精査し、ノイズを拾わないようにする。（最も安全なのは、DOMに存在する `.c_icon_exclusive` の目視確認のみを採用すること）
2. **過去データ(DB)のクレンジング**:
   - 既にDB上でAPIのノイズのせいで `is_exclusive = 1` となってしまっている過去の記事を特定し、実際のページにバッジが存在しないものは `0` にリセットする。
3. **WordPress上のタグの撤去**:
   - DBだけ直しても、WP上には誤って付与された「FANZA独占」等のタグが残ってしまうため、誤付与された作品からそのタグを引き剥がす（`wp post term remove`等のバッチ）。
4. **サイドバーの復元**:
   - 上記のデータクレンジングが完了した後、`functions.php` のコメントアウトを外し、再び正常な専売ナビゲーションを有効化する。
