# インフラ大移行録：ConoHa OOM障害からのXSERVER (KUSANAGI) 完全移行ロードマップ

## 1. 障害の背景と経緯（発端）

*   **発生事象**: 稼働中のConoHa VPS環境において、自動投稿エンジン等による突発的なメモリスパイク（OOM: Out of Memory）が頻発。
*   **致命的インシデント**: 某日未明より、ConoHa側の大規模なストレージ・ネットワーク障害が発生し、**14時間以上にわたるサーバーの完全ダウン**（コントロールパネルからの操作も不可、起動ボタンすら機能しない状態）に陥る。
*   **初動の自衛判断**: 障害の兆候（異常なエラーログ）を検知した時点で、**速やかに手動でサーバーの電源をOFF（シャットダウン）**し、データベースやファイルシステムの破損（二次災害）を未然に防ぐ「インフラエンジニアとしての完璧な防御姿勢」をとった。

---

## 2. 経営判断：新インフラ（XSERVER VPS）の選定理由

ConoHaの復旧見込みが立たない中、将来的なOOMの根絶と圧倒的なパフォーマンス向上を見据え、即断即決で**XSERVER VPS（6GBプラン）**への移行を決定した。

### 選定の決め手（なぜXSERVERだったのか）
1.  **潤沢なメモリ（6GB）による「心の余裕」**
    *   従来の数GBプランでは自動化スクリプトの実行時にOOM（メモリ不足によるプロセス強制終了）の恐怖が常に付き纏っていたが、6GBの圧倒的余裕により、精神的・システム的な安定を確保。
2.  **NVMe SSDによる圧倒的I/O速度**
    *   旧世代のSSDとは一線を画す「国内最速クラスのNVMeストレージ」を標準搭載しており、大量の画像処理やDBアクセスを行うWordPress環境に最適。
3.  **KUSANAGI 9 (AlmaLinux 9) の超高速環境**
    *   単なるLAMP環境ではなく、世界最速クラスのWordPress実行環境「KUSANAGI（魔改造チューニング済）」がテンプレートとしてワンクリックで導入可能。ページ表示速度の体感値が劇的に向上する。
4.  **長期的なコストパフォーマンス**
    *   キャンペーン適用による12ヶ月一括契約により、月額換算でランニングコストを大幅に抑えつつ、エンタープライズ級のスペックを手に入れた。

## 3. XSERVER (KUSANAGI) 設定と移行ロードマップ

ConoHaからのデータ救出および、XSERVERでの新環境構築は以下のフェーズで進行する。

### Phase 1: データの完全救出（サルベージ）
*   **前提条件**: ConoHaのコントロールパネル上で「起動（緑ランプ）」が確認でき次第、速やかに作業開始。
*   **作業内容**:
    1.  SSH接続にて旧サーバーへログイン。
    2.  WordPressの巨大な画像データ群（`wp-content/uploads/`等）と、MySQLデータベース（`.sql`ダンプ）を圧縮。
    3.  `scp` もしくは `rsync` を用いて、データを安全なローカルまたは次期サーバーへ一時退避。

### Phase 2: 新環境（XSERVER）の初期アーキテクチャ構築
*   **OS/環境**: `AlmaLinux 9` + `KUSANAGI 9 (WordPress)`
*   **初期設定手順**:
    1.  `root` パスワードの安全な設定とSSHポートの変更・鍵認証の導入（セキュリティ層の構築）。
    2.  `kusanagi init` コマンドによるミドルウェア（NGINX, MariaDB, PHP 8.x）の初期最適化。
    3.  `kusanagi provision` による指定ドメインのプロファイル作成と、空のWordPress環境の立ち上げ。

### Phase 3: データ流し込みと動作検証（リストア）
*   **作業内容**:
    1.  退避しておいたMySQLダンプデータを、新環境のMariaDBへインポート（`kusanagi` 専用コマンドによるDB最適化込み）。
    2.  画像ファイルおよびテーマ・プラグイン一式を新サーバーのドキュメントルートへ展開。
    3.  ホストマシンの `hosts` ファイルを一時的に書き換え、外部公開前に新サーバーIPでのWordPress表画面および管理画面の完全動作チェックを実施。

### Phase 4: DNS切り替えと本番稼働（グランドオープン）
*   **作業内容**:
    1.  ドメイン管理画面にて、ネームサーバー（DNSレコード）のAレコードを、新しいXSERVERのIPアドレス（ `210.131.218.83` ）へ変更。
    2.  DNSの浸透（プロパゲーション）を待ち、全世界からのアクセスが爆速のXSERVER（KUSANAGI環境）へ向くのを確認してミッションコンプリート。

## 4. 総括とインフラ的知見

本件から得られた最大の教訓は、**「異常を察知した際の『適切な電源OFF（二次災害の抑止）』」**と、**「サンクコスト（既に払ってしまった休眠サーバーの費用）に囚われず、自費で最強の環境を即座に確保する『迅速な経営判断』」**である。

いかに優れたコード（自動投稿エンジン）を書いても、土台となるインフラが貧弱（または不安定）であれば、すべては砂上の楼閣となる。本移行により、Nexusシステム等の巨大な自動化エンジンの基盤として、最強かつ堅牢な「城」が完成することとなる。
