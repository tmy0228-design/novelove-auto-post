# Novelove Nexus システム 統合マスター計画書 (v6.0)

本ドキュメントは、Noveloveダッシュボード（Nexusシステム）の全体アーキテクチャおよび、**今後の拡張に向けた実装計画（ロードマップ）を統合した唯一のマスター仕様書**です。

---

## 🧭 1. Nexus システムの基本原則
1. **検知は自動、実行はユーザー主導**: あらすじ更新や死に記事の検知はシステムが自動で行うが、リライト・パージなどの破壊的操作はユーザーが確認画面を経て明示的に実行する。
2. **即実行モデル**: リライトにキューイングは使わない。
3. **既存機能への無破壊**: `auto_post.py` の基本投稿フローへの影響は最小限に。
4. **desc_score は不変**: 記事自体の品質スコアではなく、あくまで「取得元のあらすじ充実度」を表す値として扱う。

## ✅ 2. 構築済み機能（本番稼働中）

### フェーズ1: 自動セール検知＆売れ筋タグ管理 (`nexus_revive.py`)
- FANZA/DLsite/DigiKet を12時間間隔で巡回。
- 割引率30%OFF以上に「🔥期間限定セール」タグ付与。
- ランキング上位に「🏆売れ筋作品」タグ付与。
- リライト時のタグ一元化機構において、これらセールタグが欠落しない強力な保護ロジックを実現。

### フェーズ2: リライト基盤 ＆ ダッシュボード (`nexus_dashboard.py`, `nexus_rewrite.py`)
- **一覧＆可視化**: 3大サイトの全DB記事状態を統合ビューで表示。
- **即時手動リライト**: ユーザー指定のライター・感情モードでAPIを叩き、SEOタグやメタを再構築してWPへ即時反映。
- **あらすじ更新検知**: 取得元でのあらすじ変化を差分ビューで可視化。

### フェーズ3: Google Search Console (サチコ) 連携 (`nexus_gsc.py`)
- 直近30日のインデックス状況・表示回数・クリック数を自動取得。
- **Lv1**: 未インデックス、**Lv2**: 表示0、**Lv3**: クリック0 の「死に記事アラート」を毎日Discordへ通知。

## ⛔ 3. ボツ（廃案）になった機能

1. **「強い記事」へのリアル口コミ追加（スクレイピング要約）**
   - 理由: アフィリエイト規約上、公式提供API以外の手段でのスクレイピング（口コミ収集）がアカウント停止（一発BAN）の対象となるため。
2. **完成品タグ（wp_tags）のGUI一括管理機能**
   - 理由: 現在のAIプロンプトは表記ゆれ（例：幼なじみ/幼馴染）がシステム上ほぼ発生しないよう厳密に制御しているため、専用の表記ブレお掃除機能は不要と判断。

## 🚀 4. 今後実行するタスク（本ドキュメントの主眼）

ここから先は、今後順次実装を進める **未実装の確定タスク** です。

### TASK 1: FANZAトレンド抽出 ＆ AIタグ最適化システム
これまで `auto_post.py` に固定で直書きされていたAIの判定用タグ30項目（BL 15個、TL 15個）を、ダッシュボードからトレンドに合わせて動的に入れ替えられるようにする仕組み。
- **STEP 1**: タグ保存用設定ファイル（config等）を作成し、現在の固定タグ30個を移行。
- **STEP 2**: ダッシュボードに「📈 トレンド＆タグ管理」タブを新設。
- **STEP 3**: FANZA公式API等を利用し、最新の人気キーワードを取得して「おすすめトレンド予測」として画面右に提示。
- **STEP 4**: トレンドに合わせたタグ入れ替えをUI上から行い、即座に次回の自動投稿へ反映させる。

### TASK 2: 死に記事の手動パージ（完全削除）機能
GSC連携によって特定された「死に記事」をUI上から一撃で安全に削除する機能。
- **機能**: ダッシュボードの「死に記事アラート」欄に【パージ】ボタンを追加。
- **処理**: ボタン押下でWP上から記事を完全削除（REST APIまたはWP-CLI使用）し、DBステータスを `status='deleted'` へ変更。関連記事の内部リンクはYARPP側が自動解決するため手動修復は不要。

### TASK 3: 重複記事のファジースキャン＆手動パージ
既存の Fuzzy Matching（類似度85%以上）をすり抜けてWPに二重投稿されてしまった同一・類似作品をあとから掃除する機能。
- **機能**: DB内の全公開記事（タイトル/あらすじ）を非同期または巡回処理でクロス集計・スコアリングし、重複の疑いがあるペアをダッシュボードにリスト化。
- **処理**: 人間が目視確認し、不要な方を「パージ」ボタンで安全に一撃削除できるようにする。
