# 自動コラム（まとめ）生成システム「ノベラブ・キュレーター」設計草案 (v21.0.0)

このドキュメントは、ノベラブにおける「同一タグの作品をまとめたまとめ記事（キュレーションコラム）」を安全かつ効果的に自動生成するための設計書です。
Googleの「Helpful Content System」や「EEAT（専門性・経験）」の基準に準拠し、スパムや重複判定を完全に回避しながら、売上の最大化を狙います。

> [!IMPORTANT]
> **自己レビュー実施済み (2026-06-25)**: 草案 v21.0.0-draft に対して7項目の改善を反映。
> 主な変更点: フォーマット差別化・作品数5件化・タグ選定多様化・プロンプト品質強化。

> [!WARNING]
> **本ドキュメントは実装前の草案です。** 実装済みの正式仕様は `SPECIFICATIONS.md` 第6章 §6-4 を参照してください。
> 本草案の一部の設計（タイプ分類バッジ・ランキング記事へのリンク等）は実装時に簡略化されています。

## 1. システムの目的と背景

現在のノベラブは、個別の作品紹介記事と、システムが動的に生成するタグページ（一覧）で構成されています。
ここに「同一テーマ（タグ）の複数作品を横断的に紹介するまとめ記事（コラム）」を自動的に追加することで、以下の効果を狙います。

1. **SEO（検索エンジン評価）の底上げ**: 関連する記事群を相互リンクで結ぶ「トピッククラスターモデル」を構築し、特定テーマに関するサイトの専門性（トピックオーソリティ）をGoogleに強くアピールします。
2. **回遊性と購買率（CVR）の向上**: 単なる一覧ではなく、「こういう気分の人にはこれ」という具体的な推薦と比較を提示することで、読者の購入意欲を刺激します。

---

## 2. SEO（検索エンジン対策）の設計

### A. 重複コンテンツ（コピペ）の完全回避

個別作品の紹介文をそのままツギハギすると、検索エンジンから重複・低品質コンテンツとみなされます。

- **解決策 (EEATの動的付与)**:
  - 記事の**導入コラム**を、ノベラブの公式レコメンダー1名によるコラム形式（吹き出し2〜3往復）でAIに新規生成させます。
  - 各作品紹介は「なぜこのテーマでおすすめなのか」という**テーマ特化の切り口**で個別に新規執筆し、既存の個別記事あらすじとは別の文章を生成します。

### B. キーワードのカニバリゼーション（自社競合）対策

「タグページ」「まとめ記事」「ランキング記事」の3種が同一キーワードで競合しないよう、タイトルと構成を明確に棲み分けます。

| コンテンツ種別 | タイトル例 | 狙うキーワード | 構成 |
|:---|:---|:---|:---|
| タグページ（自動） | `逃げられない愛『執着』BL/TLおすすめ紹介｜2026` | 網羅・カタログ | カード一覧 |
| **まとめ記事（本システム）** | `【5選】愛が重すぎる…！執着BLコミック・小説おすすめ比較` | 比較・厳選・〇選 | **コラム＋比較表** |
| ランキング記事（既存） | `DLsite BL ランキング 2026年6月 第4週` | 週間・ランキング | 吹き出し対話形式 |

> [!NOTE]
> まとめ記事は**コラム＋比較表形式**を採用し、既存のランキング記事（吹き出し対話形式）と明確に差別化します。
> これにより、Googleが「異なるコンテンツタイプ」として認識しやすくなります。

### C. 内部リンク構造の最適化（トピッククラスター）

- **まとめ記事 → 個別記事**: 各作品の紹介から、ノベラブ上の個別紹介記事へアンカーテキスト付きリンク
- **まとめ記事 → タグページ**: 末尾に「他の[タグ名]作品一覧はこちら →」のリンクを設置
- **まとめ記事 → ランキング記事**: 末尾に同ジャンルの直近ランキング記事へのリンクを自動挿入（例: `📊 最新のDLsite BLランキングはこちら →`）

### D. 構造化データ（JSON-LD）の自動挿入

`functions.php` で実装済みの `ItemList` JSON-LD ロジックを、`post_type = 'curation'` の記事にも発火するよう拡張します。
Googleリッチリザルト（「〇〇 おすすめ」検索時のリスト表示）の獲得を狙います。

## 3. ユーザー体験（UX）と購買（CVR）の設計

### A. 読者の選択を助ける「5択ロードマップ」

5つの作品を以下のタイプに自動分類して提示します。分類はDBのタグデータから自動判定し、分類不能な場合はAIに委任します。

| タイプ | 判定方法 |
|:---|:---|
| **初心者向け** | `gsc_clicks` が最も多い作品（多くの人が読んでいる定番） |
| **甘々・ハッピーエンド** | `ai_tags` に「溺愛」「甘々」「ハッピーエンド」「純愛」等を含む |
| **執着・ダーク** | `ai_tags` に「執着」「束縛」「ダーク」「闇落ち」「独占欲」等を含む |
| **ボイス・音声** | メディア種別が「voice」の作品 |
| **編集部イチ押し** | `desc_score` が最高値の作品 |

これにより、読者は「今の気分に合うもの」を直感的に選択できます。

### B. 比較テーブル（データ表）の自動挿入

記事の冒頭コラム直後に、紹介する5作品の比較表をHTMLテーブルで自動出力します。

| 項目 | 内容 |
|:---|:---|
| タイトル | 作品名（個別記事リンク付き） |
| 評価 | `desc_score`（★表示） |
| おすすめタイプ | 上記5分類のラベル |
| 配信サイト | DLsite / FANZA / らぶカル |
| メディア | 漫画 / 小説 / ボイス |

## 4. 自動選定アルゴリズム（カニバリ・重複の完全排除）

毎週水曜 11:00 JSTに自動実行されるバッチ処理（`novelove_curator.py`）で動作します。
履歴管理のために novelove_posts テーブルに post_type='curation' として保存します。

### A. 対象タグの選定（月4本）

毎週1本のペース（月4本）で、以下のルールに基づきタグを決定します。

| 週 | 選定方法 | 目的 |
|:---|:---|:---|
| 第1〜3週 | **タグの合計GSCクリック数が少ない順**（フィルター通過後） | 埋もれたタグの底上げ |
| 第4週 | **クロスタグ**（2タグの組み合わせ・同条件フィルター適用） | ニッチ×ニッチで差別化 |

**共通フィルター条件**:
1. そのタグが付与されている公開記事が「10件以上」あること
2. 過去「90日間」に、そのタグでのまとめ記事が生成されていないこと（クールダウン）

> [!NOTE]
> **まとめ記事の目的はサイト全体のSEO底上げ。** GSCクリックが少ないタグ＝埋もれた記事群を、まとめ記事からの内部リンクで引き上げることが主目的。強い記事はすでにGoogleに評価済みのため内部リンクの恩恵が小さく、弱いタグに集中することでサイト全体のPageRankが底上げされる。

**第4週クロスタグの実績（本番DB調査済み・2026-06-25）**:
- 5件以上の組み合わせ: **184通り**（月1本ペースで15年以上のネタ量）
- DeepSeekがタイトル候補・導入切り口・作品ミニレビューをデータから自動生成
- 注目例: ヤンデレ×執着(155件) / 転生×執着(55件) / スパダリ×執着(62件) / ASMR×バイノーラル(35件)

### B. 掲載作品（5件）の選定

1. 対象タグが紐付いている公開済み記事
2. **作品クールダウン**: 過去「120日間」に他のまとめ記事に掲載されていないこと
3. 1〜2を満たす作品から、セクション3-Aの5タイプ分類を優先しつつ5件を自動抽出

> [!NOTE]
> `desc_score` によるフィルターは不要。公開済み記事はすべて `desc_score` 4以上の品質基準を満たしているため。

## 5. 記事フォーマットとAI生成プロンプトの骨子

### 記事構成（コラム＋比較表形式）

```
[導入コラム]
  レコメンダー1名による吹き出しトーク（2〜3往復）
  ・テーマの魅力・今回の選定理由を語る（計200文字程度）

[比較テーブル]
  5作品の一覧表（タイトル・評価・タイプ・サイト・メディア）

[各作品の個別紹介]（5件分）
  ・h3見出し: 作品タイトル
  ・おすすめタイプバッジ（例: 🌸 初心者向け）
  ・テーマ特化のミニレビュー（60文字程度）
  ・個別記事へのリンクボタン

[フッターリンク]
  ・タグページへの誘導リンク
  ・直近のランキング記事への誘導リンク
```

### AI生成プロンプトの骨子

以下のルールは `novelove_ranking.py` および `novelove_writer.py` と完全に統一する。
実装時は `novelove_soul.py` の定数をそのままインポートして使用すること。

**【キャラクター設定】**
- `REVIEWERS` からジャンル対応レコメンダーを選定（`_get_reviewer_for_genre` と同じロジック）
- `MOOD_PATTERNS` からランダム選択した感情モードを冒頭に適用（毎回違うトーンにする）
- 挨拶ランダム化（10%確率で身の上話あり・90%確率で禁止）← ランキング記事と同じ

**【入力データの前処理】**
- 各作品の説明文は `mask_input(description, level=1)` でマスク処理してからプロンプトに渡す
- 説明文は300字以内に切り詰めて渡す（トークン節約・ランキング記事と同じ）

**【執筆ルール（全記事共通・省略禁止）】**
1. raw HTMLのみを出力。\`\`\`やコードブロックは使わないこと
2. 直接的な性的単語（性器の名称・行為の直接名称）は使用禁止。官能的な比喩を使うこと
3. メディア中立表現：漫画・小説・音声が混在するため「読む」「聴く」「本を開く」等の
   メディア特定表現は禁止。「楽しむ」「チェックする」「体験する」等の中立表現のみ使用
4. 作品説明文はコピペ禁止。テーマの視点に特化した新規文章を生成すること
5. `FACT_GUARD` 適用：あらすじに書かれていないキャラ名・展開・結末の創作は絶対禁止。
   「読み終えて」「読了後」等の読了済みを装う表現も禁止
6. `NG_PHRASES` 適用：「いかがでしたか」「ぜひチェックしてみてください」等のAI的無難フレーズ禁止
7. 吹き出しHTMLは `speech-bubble-left` クラスのみ（まとめ記事はレコメンダー1名形式のため）

**【出力指示】**
1. 導入コラム（吹き出し2〜3往復・計200文字程度）
   - テーマの魅力と今回の選定理由をキャラの口調で語る
   - 冒頭の書き出しは毎回違う切り口にすること（マンネリ禁止）
2. 各作品の紹介（5件 × 60文字程度）
   - 「なぜこの作品がこのテーマでおすすめなのか」をテーマ特化の切り口で新規執筆
   - あらすじの事実のみに基づくこと（捏造禁止）
