# 自動コラム（まとめ）生成システム「ノベラブ・キュレーター」設計草案 (v20.7.0-draft)

このドキュメントは、ノベラブにおける「同一タグの作品をまとめたまとめ記事（キュレーションコラム）」を安全かつ効果的に自動生成するための設計提案書です。
Googleの「Helpful Content System」や「EEAT（専門性・経験）」の基準に準拠し、スパムや重複判定を完全に回避しながら、売上の最大化を狙います。

---

## 1. システムの目的と背景
現在のノベラブは、個別の作品紹介記事と、システムが動的に生成するタグページ（一覧）で構成されています。
ここに「同一テーマ（タグ）の複数作品を横断的に紹介するまとめ記事（コラム）」を自動的に追加することで、以下の効果を狙います。
1. **SEO（検索エンジン評価）の底上げ**: 関連する記事群を相互リンクで結ぶ「トピッククラスターモデル」を構築し、特定テーマに関するサイトの専門性（トピックオーソリティ）をGoogleに強くアピールします。
2. **回遊性と購買率（CVR）の向上**: 単なる一覧ではなく、「こういう気分の人にはこれ」という具体的な推薦と比較を提示することで、読者の購入意欲を刺激します。

---

## 2. SEO（検索エンジン対策）の設計

### A. 重複コンテンツ（コピペ）の完全回避
個別作品の紹介文をそのままツギハギすると、検索エンジンから重複・低品質コンテンツとみなされます。
- **解決策 (EEATの動的付与)**:
  - 記事の導入コラムを、ノベラブの公式レコメンダー（例：もも＆レイなど）2名による「ミニ対談・掛け合い」の形式で、AI（DeepSeek/Gemini）にその場で完全新規で生成させます。
  - レコメンダーA（ライト層・初心者向けの視点）とレコメンダーB（コア層・マニア向けの視点）の掛け合いにすることで、ネット上に存在しない独自の一次情報を創出します。

### B. キーワードのカニバリゼーション（自社競合）対策
自動生成の「タグページ（一覧）」と「まとめ記事」が同じキーワードで競合しないよう、タイトルや見出しのルールを明確に定義して棲み分けます。
- **タグページ（網羅・カタログキーワード）**:
  - タイトル例: `逃げられない愛『執着』BL/TLおすすめ紹介｜2026`
- **まとめ記事（比較・厳選キーワード）**:
  - タイトル例: `【厳選】愛が重すぎる…！『執着』属性が心に刺さるBLコミック・小説3選`
  - 「厳選」「3選」「コミック・小説」などの購入検討キーワードを含めることで、ユーザーの検索意図に合わせて表示を切り分けます。

### C. 内部リンク構造の最適化
- **個別記事への流入**: まとめ記事から各作品の個別紹介記事へ、文脈に沿ったアンカーテキスト（作品名など）で直接リンクを張ります。
- **タグページへのリンク集約**: まとめ記事の末尾から、親となるタグページへ「他の[タグ名]作品一覧はこちら」としてリンクを戻します。

## 3. ユーザー体験（UX）と購買（CVR）の設計

### A. 読者の選択を助ける「3択ロードマップ」
3つの作品をただ並べるのではなく、読者の好みに合わせて以下のようにおすすめタイプを自動で割り振ります。
- **作品1**: 「初心者向け（まず手にとるならこれ！）」
- **作品2**: 「マモル・甘々向け（心満たされるハッピーエンド）」
- **作品3**: 「ハード・過激向け（強い刺激と執着を求める人へ）」
これにより、読者は「今の気分に合うもの」を直感的に選択できます。

### B. 比較テーブル（データ表）の自動挿入
記事のメイン部分に、紹介する3作品の以下のデータを整理した「比較表」をマークダウンテーブルで自動出力します。
- 項目: タイトル / 評価（desc_score）/ 価格 / 配信サイト（DLsite、FANZA等）/ メディア形式（音声、漫画、小説）
ユーザーにとって「一目で比較できる有益な情報」を配置し、ページの価値を高めます。

## 4. 自動選定アルゴリズム（カニバリ・重複の完全排除）

毎週1回（例：木曜夜）に自動実行されるバッチ処理（`novelove_curator.py`）で、以下のロジックに基づき「今書くべきテーマ」と「紹介すべき作品」を自動選定します。履歴管理のために `curation_history` テーブルをDBに新設します。

### A. 対象タグの選定
1. **記事件数**: そのタグが付与されている公開記事が「10件以上」あること。
2. **クールダウン期間**: 過去「90日間」に、そのタグでのまとめ記事が生成されていないこと。
3. 1と2を満たすタグの中から、Google Search Console（GSC）で最近のクリック数（または表示回数）が伸びている「急上昇タグ」を1つ自動決定。

### B. 掲載作品（3件）の選定
1. 対象タグが紐付いている作品。
2. **作品のクールダウン**: 過去「120日間」に、他のまとめ記事に掲載されていないこと（特定の作品ばかり露出するのを防ぐ）。
3. **品質担保**: 品質スコア（`desc_score`）が「4以上」の高品質な記事。
4. 1〜3を満たす作品から、直近のアクセス数やランダム性を用いて3件を自動抽出。

## 5. AI生成プロンプトの骨子
AI（DeepSeek-V4-Flash / Gemini）にコラムを生成させる際の主要な指示書テンプレートです。

```text
あなたは『ノベラブ』の公式レコメンダー（キャラクター属性を忠実に再現）です。

【入力データ】
- 対象テーマ（タグ）: [タグ名]
- レコメンダーA: [レコメンダー名1]
- レコメンダーB: [レコメンダー名2]
- 作品A: [タイトルA] - [特徴・キーワード]
- 作品B: [タイトルB] - [特徴・キーワード]
- 作品C: [タイトルC] - [特徴・キーワード]

【出力指示】
1. 導入部:
   指定されたテーマについて、レコメンダーAとBの2人による掛け合い対談の形で、そのテーマの魅力や今回の選定理由を語る150文字程度のイントロコラムを作成してください。
2. 各作品の紹介:
   元の紹介文をコピペせず、今回のテーマ（例：執着）の視点に特化し、「なぜこの作品がこのテーマでおすすめなのか」をそれぞれ80文字程度で新規に紹介してください。
```

## 6. 実装に向けた開発タスク（予定）
本システムを本番に導入する際の実装手順です。
1. **DBマイグレーション**: `curation_history` テーブルの作成
2. **コアロジック作成**: `novelove_curator.py`（選定・AI生成・WP自動投稿を担うバッチ）の新規実装
3. **プロンプトテスト**: 各タグと作品を用いたAI生成コラムの検証
4. **cron登録**: 週次で `novelove_curator.py` を実行する設定の追加
