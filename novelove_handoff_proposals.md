## 🏆 プラットフォーム別・戦略的ポジショニング（合意済み基本方針）

本番のGoogle Search Console（GSC）データ分析に基づき、以下の役割分担を戦略として確定させています。今後のすべての施策はこの前提で行います。

1. **DLsite（超・強化 ➔ サイトの絶対的エース）**
   - **現状**: 検索流入の約87%を占めており、作品名での競合が極めて少ないブルーオーシャン。
   - **方針**: すでに強い絶対的エースを「さらに強く」する。
   - **役割**: 個別記事名で検索上位を独占し、アクセス流入を稼ぐメインの入り口、かつ最大の「収益化の出口（ハブ）」として育成する。
2. **らぶカル（育成・強化 ➔ 高単価なサブ主力）**
   - **現状**: 同人作品だがライバル自動ブログが多く、露出はDLsiteの4分の1。ただし成約単価が高い。
   - **方針**: 将来性のあるポテンシャル株として「意図的に育てる」。
   - **役割**: 個別記事から「らぶカルランキング（固定URLハブ）」へ優先的に内部リンクを集約し、ドメイン評価をこのハブ記事に集中させて競合ブログに競り勝つ。
3. **DMM商業（他力本願・底上げ待ち ➔ おまけコンテンツ）**
   - **現状**: 大手電子書籍ストアや老舗レビューブログなど競合が強すぎ（超レッドオーシャン）、個別作品名でも検索結果にほぼ表示されない。
   - **方針**: コスパが悪いため、単体でのSEO強化やライター改修は行わない。
   - **役割**: 自動投稿によるロングテールで稀に数PVを拾う「おまけ」としての扱いに留める。DLsiteやらぶカルが強くなった結果、つられて順位が上がってくるのを自然に待つ。

## 💡 今回の会話で決定・実装した事項 (v21.4.0)

### 注目作品5件のPHP完全静的化と配置（シンプルイズベスト）
- **表示順とUXの維持**: 
  複雑なJavaScriptによる要素のDOM移動などはバグの原因になるため一切行わず、**「PHPのみで完結するシンプルイズベスト」**の設計を適用。
- **配置位置の決定**: 
  本番のHTML構造を正確に解析した上で、**「アフィリエイトクレジットのすぐ下、関連記事（YARPP）の上（属性タグのすぐ下）」**を表示位置として決定。
- **実装方法（優先度11フィルター）**:
  `functions.php` の中で、本文（`the_content`）の末尾に優先度10で自動結合される属性タグ（`post-tags-container`）の直後に、**優先度「11」**で人気記事5件のHTMLを静的結合して出力。これにより、YARPPテンプレートを汚すことなく、完璧に「関連記事のすぐ上」に静的HTMLとして直書きで出力される安全な仕組みを構築しました。
- **重み付けとクォータ制の導入**:
  `nexus_gsc.py` にて、GSCの単純クリック順ではなく「収益重み付け（DLsite専売2倍、通常1.5倍、らぶカル1.5倍、DMM0.5倍）」を適用。かつ特定の販売サイトで埋め尽くされないよう「DLsite最大5件、DMM最大2件、らぶカル最低2件優先確保」のプラットフォーム枠制限（クォータ制）をプログラム制御し、WordPressに自動同期するバッチを完了。

## 🚀 今からすること（未着手の次回改善タスク）

### アフィリンク送客率の計測

**目的**:
ASP側の管理画面で追跡できない「どの記事が何回アフィリンクをクリックされたか（送客されたか）」を自サイト内で完全に計測・可視化し、記事タイプ別の「送客率（アフィクリック数 / GSCクリック数）」をダッシュボードで分析できるようにする。

**背景**:
ASP成果と個別記事の紐付けは不可能であるため、手前のアフィリンク「クリック数」を計測し、GSC of 流入クリック数と掛け合わせて「送客率（CVR）」を測定するのが、アフィリエイトの改善における現実的な最重要指標となる。

**詳細設計・開発要件**:
1. **アフィリンクへのデータ属性の仕込み**:
   `novelove_core.py` 内の `get_affiliate_button_html()` や、まとめ記事・ランキング記事の生成コードにおいて、`<a>` タグへカスタムデータ属性を自動付加する。
   ```html
   <a href="{url}" class="novelove-aff-link" data-pid="{pid}" data-site="{site}" data-genre="{genre}" data-type="{post_type}">
   ```
2. **ブラウザ遷移を妨げない非同期ログ送信**:
   テーマのフッター等に軽量なJavaScriptを埋め込み、`.novelove-aff-link` クラスのクリックイベントをキャッチ。`navigator.sendBeacon()` を用いて非同期でバックグラウンド送信し、ユーザーの遷移速度に悪影響を与えずにデータをWordPressの REST APIカスタムエンドポイント（`/wp-json/novelove/v1/click`）へ投げる。
3. **クリックログテーブルの新設**:
   SQLite（`novelove_unified.db`）内に `novelove_clicks`（`click_id`, `product_id`, `site`, `genre`, `post_type`, `clicked_at`）テーブルを作成し、ログデータを蓄積する。
4. **ダッシュボード（nexus_dashboard.py）への表示**:
   ダッシュボード上に「プラットフォーム別送客率」「記事タイプ別送客率」のグラフ・表を追加し、質の高い送客ができている記事グループを視覚的に浮き彫りにする。
