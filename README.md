# Novelove 自動投稿システム (v8.4.0)

> [!IMPORTANT]
> **開発・修正時の注意**: このリポジトリのコードは複数の場所から随時修正が行われます。作業を開始する前に、必ず GitHub で最新版（`main` ブランチ）を確認・プルしてから最新のコードに対して作業を行ってください。

Novelove（ノベラブ）のコンテンツ生成および自動投稿を行うエンジン一式です。
高品質な**日本語マンガ**に特化したレビュー記事を全自動で生成・投稿します。

## 🚀 システム概要
- **目的**: FANZA, DLsite, DMMブックスの新着情報を取得し、DeepSeek AIでレビュー記事を生成して WordPress に自動投稿する。
- **主要言語**: Python 3.x
- **データベース**: SQLite (`novelove.db`, `novelove_dlsite.db`)

## 🛠 主要機能・ガード (v8.4.0)
高品質なサイト運営のため、多層的なフィルターとガードを実装しています。

### 1. ハイブリッド順次投稿システム (1時間1件厳守)
API費用の節約と1時間1件のペースを厳守するため、以下の優先順位で動かします。
- **新着枠**: 画像があり、発売7日以内の新作を最優先審査。4点以上で即投稿。
- **保険枠 (行列消化)**: 新着がない場合、DB内の「未審査」から発売日が最古のものを1件再審査。画像なしなら即除外、画像あり4点以上で投稿。

### 2. APIコスト最適化ガード
AI審査（DeepSeek）を呼ぶ前に以下のチェックを行い、不要なコストを100%カットします。
- **画像チェック**: 「NOW PRINTING」や「No Image」等の場合は審査をスキップ。
- **発売日チェック**: 発売まで8日以上ある作品は、画像が出揃うまで審査を保留。

### 3. ハイブリッド・ノイズフィルター
ボイス作品、ノベル、ゲーム、外国語版などの混入を徹底的に排除します。
- **DLsite**: 商品詳細ページの `.work_genre` から公式の `MNG`（マンガ）バッジを直接チェック。
- **FANZA**: 作品形式（コミック・劇画）に加え、タイトルの特定の括弧パターン（【韓国語版】等）で精密照合。

## ⏰ 定期実行設定 (cron)
サーバー上で以下のcronを設定することで、定期的に自動投稿が実行されます。
```bash
# 例: 3時間おきに実行
0 */3 * * * cd /home/kusanagi/scripts && /opt/kusanagi/bin/python3 auto_post.py >> /home/kusanagi/scripts/novelove.log 2>&1
```
