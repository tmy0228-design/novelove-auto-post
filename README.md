# Novelove 自動投稿システム

Novelove（ノベラブ）のコンテンツ生成および自動投稿を行うエンジン一式です。
Claude や Gemini などの AI アシスタントが協力して開発・メンテナンスを行えるよう、構成やルールを整理しています。

## 🚀 システム概要
- **目的**: FANZA, DLsite, DMMブックスの新着情報を取得し、Gemini AIでレビュー記事を生成して WordPress に自動投稿する。
- **主要言語**: Python 3.x
- **データベース**: SQLite (`novelove.db`, `novelove_dlsite.db`)

## 📂 ファイル構成
- `auto_post.py`: メインエンジン（取得・執筆・投稿・Discord通知のすべてを集約）
- `update_existing_posts_style.py`: 既存記事のスタイルを一括修正するユーティリティ

## 🛠 重要ルール（AI・開発者向け）
開発・修正を行う際は、以下の仕様を必ず維持してください。

### 1. 投稿ガード
- **画像チェック**: ダミー画像（now printing/no image等）の投稿を防ぐ厳守ルール。
- **最終防衛**: WordPress投稿直前にも `_check_image_ok` で画像の生存確認を行うこと。

### 2. 再チェック（救済）ルール
- **未発売のみ**: 発売日を過ぎた未完成の商品（画像なし等）は再チェックせず `failed_stock` とする。
- **インターバル**: サーバー負荷とAPI節約のため、1つの商品への再チェックは最低24時間空ける。

### 3. コンテンツ生成
- **モデルローテーション**: APIエラーを回避するため、複数のGeminiモデルを順次切り替えて使用する。
- **3段階フィルター**: 官能的表現による生成拒絶を防ぐため、フィルターレベルを0→1→2と上げて再試行する。

## 🤖 AI モデル構成 (v7.3.3.5)
分散処理とAPI制限回避のため、以下のモデルを使い分けています。

| 役割 | 使用モデル (優先順) | 特徴 |
| :--- | :--- | :--- |
| **審査 (Check)** | `gemini-2.5-flash` (メイン)<br>`gemini-2.0-flash`<br>`その他Flash/Lite各位` | 常に 2.5-flash を優先。全制限時は作品を保護(スキップ)。 |
| **投稿 (Pro)** | `gemini-2.5-pro`<br>`gemini-3.0-flash`<br>`gemini-3.1-flash-lite` | 高精度・高速・軽量の 3段階体制。執筆クオリティを維持。 |
