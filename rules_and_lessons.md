# 運用ルールと教訓 (Novelove Project)

## GitHub 同期と最新状態の確認について

### 【教訓】GitHub raw ファイルの挙動
GitHub の `raw` ファイル（`https://github.com/.../raw/main/...`）を fetch してコードを取得・確認する場合、**キャッシュによって古いバージョンが返ってくるリスク**があります。
「GitHub に push したはずなのに反映されていない」「古いコードで上書きされた」といった誤認の主な原因となります。

### 【鉄則】最新状態の確認手順
コードの最新状態や反映状況を確認する際は、GitHub の raw ファイルのみに頼らず、必ず **VPS サーバー上の実態**を確認してください。

1. **確認コマンド**:
   ```bash
   cd /home/kusanagi/scripts
   git log --oneline -5
   ```
2. **判断基準**:
   GitHub 上の表示ではなく、VPS 側で `git log` を叩いて得られるコミットハッシュやコメントが「真実」であると判断します。
