# Novelove 自動投稿システム 詳細仕様書 (v8.4.0)

本ドキュメントは、Novelove（ノベラブ）エンジンの現在の動作仕様をまとめたものです。システムの基本ルールや、AIによる審査基準などを詳細に記述します。

---

## 📅 1. 投稿頻度とスケジュール
- **投稿ペース**: 1時間につき最大1件。
- **実行間隔**: サーバー（Kusanagi）の cron により1時間おきに実行。
- **クールダウン**: 前回の投稿から 55分 未満の場合は、重複投稿を避けるため実行をスキップします。

## 🔄 2. 投稿選抜ロジック（ハイブリッド方式）
毎回の実行時に、以下の 2 段階の選抜フローを行い、1件の投稿作品を決定します。

### ステップ1：新着枠 (Pending)
その回に新しく取得したデータの中で、以下の条件を満たし、AI審査で **4点以上** を獲得したものを最優先で投稿します。
- 画像が存在する
- 発売日が直近（当日を含む 7日以内）である

### ステップ2：保険枠 (Insurance Slot - 行列消化)
新着枠に該当がない場合、データベース内の `watching`（未審査・待機）ステータスの作品から選抜します。
- **並び順**: 発売日が古い順（最古順）かつ、DB登録が古い順。
- **再審査**: 
    1. サイトの最新情報をスキャンし、あらすじや画像の有無を確認。
    2. 画像があり、AI審査で **4点以上** を獲得すれば合格。
    3. 不合格（1-3点）または画像なしの場合は `excluded`（除外）へ移行し、行列から取り除かれます。
- **目的**: 投稿するものがない「空き時間」に、過去の有望作品を掘り起こして投稿し、取りこぼしを防ぎます。

## 💰 3. APIコスト最適化（AI呼び出し制限）
DeepSeek API の消費を抑えるため、以下のいずれかに該当する場合は **AI 審査（有料）を行わず**、即座に判断を下します。
- **画像なし**: 「NOW PRINTING」や placeholder 画像の場合は審査せず `watching` で維持、または `excluded` へ。
- **発売日が遠い**: 発売まで 8日以上 ある場合は、情報が揃うまで審査を保留し `watching` とします。

## 📊 4. AI 審査基準（0〜5点スコアリング）
DeepSeek があらすじとタイトルを元に、以下の基準で判定します。

| スコア | アクション | 内容の定義 |
| :--- | :--- | :--- |
| **4.0 〜 5.0** | **合格 (Pending)** | 内容が魅力的で紹介価値が高い。即執筆・投稿対象。 |
| **1.0 〜 3.0** | **不合格 (Excluded)** | 内容が薄い、紹介しづらい、または一般的すぎる。除外対象。 |
| **0.0** | **除外 (Excluded)** | ボイス、小説、外国語版、その他非漫画コンテンツ。 |

## 🚫 5. ノイズフィルター仕様
以下のコンテンツは、AI審査を待つまでもなく、自動的に除外、または厳格に判定されます。
- **対象外カテゴリ**: 同人ボイス、ASMR、ボイスドラマ、小説（BL/TL小説含む）、ゲーム、CG集。
- **外国語版**: タイトルに `【韓国語版】` `【英語版】` などの特定の括弧パターンや、DLsite の言語タグがあるもの。
- **ホワイトリスト（DLsite）**: 商品詳細のジャンルバッジに `MNG`（マンガ）が含まれているもののみを執筆対象とします。

## 📢 6. 運用・通知
- **Discord通知**: 投稿に成功した際、以下の情報を通知します。
    - 投稿ルート（`✅ 新着投稿` か `🔄 保険枠・再審査投稿` か）
    - AIスコアと使用モデル名
    - 記事の文字数と累積投稿数
- **ジャンル回転**: FANZA, DMM, DLsite の各ジャンル（BL/TL）を、実行のたびにローテーションで切り替えてチェックします。
