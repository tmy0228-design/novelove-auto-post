# Novelove Nexus システム (統合ダッシュボード＆自動拡張) 全体再設計プラン

本ドキュメントは、Noveloveプロジェクトの次期開発におけるマスター設計構想です。
v11.x系で「小説ジャンルの有効化」などの一部機能が既に完了したことを受け、全体方針をアップデートしています。
また、運用負荷とAPI利用制限を考慮した大幅な最適化が施されています。

---

## 1. コア機能①: 超軽量なセール自動検知と蘇生（逆引き方式）
過去にAIが投稿し埋もれてしまった記事（死に記事）の中から、稼ぎ頭となる「セール中作品」だけをサーバー負荷ゼロで抽出・トップに再浮上させる。

### 発想の転換（逆引き・全件カバー方式）
自社DBの1万件の記事を1件ずつ通信してセール確認する（BANリスク大）のではなく、「相手のセール一覧まとめ」を引き抜いて自社DBと照合するO(1)アプローチ。

- **取得方法**: DLsiteの「割引中・セール対象作品一覧」URLや、DMM等のキャンペーン一覧APIから、セール中の全商品IDをガサっと一括取得する。
- **照合処理**: 拾ってきた「世の中のセール品IDリスト（数千件）」と、手元の「自社DBの1万件」をSQL等で突合（INNER JOIN）するだけ。
- **メリット**: 数回の通信（ページ送り分）だけで済むため、BANリスクやサーバー負荷が完全にゼロ。毎日1万記事を余裕で全件フルカバーできる。

### アクション: 安全な蘇生と鎮静化
- **セール告知**: タイトルの先頭に `【〇%OFF】` を付与 ＋ `🔥セール中` タグを追加する。※本文のHTML書き換えは事故リスクが高いため一切行わない。
- **自然なトップ浮上**: WordPressの公開日（post_date）はSEOリスクを避けるため触らず、**更新日（post_modified）のみを現在時刻に更新**する。WP側の表示設定で更新順に並べることでトップに浮上させる。
- **鎮静化**: セール終了を検知した際は、タイトルとタグからセール文言を外して元通りにする。

---

## 2. コア機能②: リンク切れ検知の完全自動化（プラグイン委任）
- **方針変更**: 自作スクリプト（Python）によるリンクの死活監視は処理が重いため**廃止**する。
- **代替策**: WordPressの超強力な既存プラグイン（`Broken Link Checker` など）に完全に役目を丸投げする。これにより自社構築システムの複雑さが激減し、堅牢性が飛躍的に向上する。

---

## 3. コア機能③: ブラックボックスの可視化ダッシュボードとセキュリティ
手動運用や確認作業をブラウザ経由で1クリックで行える「Nexus管理画面 (`nexus_app.py`)」を構築する。技術スタックは Streamlit を採用。

### メニュー・機能構成
- **🏠 ホーム**: 今日の投稿状況、在庫件数アラート、緊急停止ロックの「一発解除ボタン」。
- **📦 在庫管理**: 投稿待ち（pending）一覧の可視化。「ボツ（Exclude）」や「手動即時投稿」のワンクリック操作。
- **🔁 リジェクト再審**: AI審査落ち等の理由確認と、「別の感情モード・別のレビュアーに変更して再執筆させる」手動リトライ機能。
- **📈 パフォーマンス**: GSCアクセス実績の一覧、セール蘇生対象になった記事の履歴確認。

### セキュリティとアクセス保護
Cloudflare Zero Trust（Cloudflare Access）を導入する。
面倒なBasic認証やIP制限を避け、メールに届くワンタイムPINで、自宅・会社・スマホから安全かつ簡単にアクセス可能にする。

## 4. 追加・拡張機能群

### GSC連動スコアリング（記事の健康診断）
Google Search Console API と連携し、各記事にパフォーマンスタグを自動付与する。
- 🟢 健康: バリバリ読まれている
- 🟡 衰退: 伸び悩んでいる、リライト候補
- 🔴 死亡: 誰も見ていない

### AI トレンド自動追従（季節ブースト）
1. 月に1回、AIに「今月の女性向けコンテンツのトレンドタグ」を推測させる（例: 2月=チョコ、溺愛など）。
2. 在庫の中から、トレンドに合致するタグを持つ作品の優先度をアップさせる。これにより、運用者がトレンドを追わなくても自動で季節に合った作品が優先投稿される。

### 投稿後セルフチェック（安全装置）
記事の公開直後にシステム自身がURLにアクセスし、品質崩壊がないかチェックする。
- アイキャッチ画像の表示確認
- アフィリエイトボタンの正常動作
- 致命的なHTML崩壊
→ 失敗を検知した場合は下書きに降格し、Discordに「⚠️ 投稿後チェック失敗」として即時報告する。

## 5. 廃止・保留された旧計画
- **小説機能の有効化**: `auto_post.py` v11.x で既に実装完了しているため削除。
- **`nexus_core.py` の新設**: コードの二重管理を防ぐため廃止。共通ロジックは既存の `novelove_core.py` を再利用する。
- **X(Twitter) ワンクリック連携**: 当面は保留。将来的に X API の従量課金プランを利用する構想として残す。
