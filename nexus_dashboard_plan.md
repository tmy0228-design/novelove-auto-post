# Novelove Nexus システム (記事蘇生・ダッシュボード機能) 構想プラン

本ドキュメントは、過去にAIが投稿し埋もれてしまった記事（死に記事）を定期的に復活させ、サイト全体の鮮度とアクセスを底上げするための「一元管理・自動リライト（蘇生）システム」の設計構想です。

## 1. 企画の背景：なぜ「Nexus（蘇生）」が必要か？
現在の `auto_post.py` では新着記事が毎日インデックス作成されていきますが、時間経過とともに「死に記事」化し、トップページから押し出されるという課題があります。
**Nexusシステム** は、通常投稿のスクリプトとは完全に独立して定期稼働し、WordPress上の既存記事を巡回して「再浮上のフック（セールなど）」を見つけ出すシステムです。

---

## 2. コア機能①: セール自動検知と記事の蘇生（トップへの引き上げ）

### 直近のセール情報の取得と判定
1. **定期巡回スクリプト**:
   - WPデータベース内の既存の投稿済み記事のリスト（`product_id`）を読み込む。
   - 週に数回など、深夜などの低負荷帯に FANZA / DLsite などの API を叩き、既存記事の「最新価格・セール情報」を再取得する。
   
2. **セール感知（Trigger）**:
   - キャンペーンや割引（価格の変動）を検知した場合、その記事を「蘇生対象」としてピックアップ。

### リライトと露出の最適化（アクション）
1. **タグ・カテゴリの自動付与**:
   - 記事に特製の「🔥現在セール中」などのタグを自動付与する。
   - サイト上の特別な「セール対象作品カテゴリ」に一時的に記事を移動、または追加登録する。
   
2. **更新日（公開日）の変更とトップ露出**:
   - WordPress の「公開日（post_date）」そのものを現在時刻に更新（WP-CLIによる `post_date` 更新）。
   - これにより、数ヶ月前の古い記事が **サイトのトップページ（最新記事一覧）** の先頭に再び躍り出ることになる。
   - 同時に Google クローラーに対しても「ページが大幅に鮮度を取り戻した」と強力なシグナルを送る。

3. **本文リッチ化（任意）**:
   - 本文の冒頭に「※現在〇〇%OFFセール中！（〇月〇日まで）」といったセール告知文言を自動追記。

---

## 3. コア機能②: セール終了後の自動後片付け（鎮静化）

セールが終了したのに「セール中」と謳ったままになるのは読者の信頼を損ねます。そのため、元の定価に戻った際の後片付けフローも完全に自動化します。

1. **セール終了の検知**:
   - 巡回API通信時に「通常価格に戻っている」または「販売終了状態」と判定された記事をピックアップ。
2. **情報の巻き戻し（後処理）**:
   - 「🔥現在セール中」タグ、および「セールカテゴリ」から対象記事を外す。
   - 冒頭に自動追記したセール告知文言の内容を削除（元の状態に戻す）。
   - （※更新日や公開日は据え置きで良く、新しい新着記事に押し出される形でふたたび古い順に後方へ沈んでいくのに任せる）

---

## 4. コア機能③: ブラックボックスの可視化ダッシュボード

この「蘇生システム」や「通常投稿システム」の稼働状況を、WordPress管理画面やWebブラウザから一目で確認できるダッシュボード（Nexus管理画面）を構想します。

**【可視化する指標・機能】**
- **在庫状況の監視**: 現在稼働中の各サイトごとの未投稿在庫数（pending）が一目でわかるゲージ表示。
- **蘇生ログ**: 今週のセール検知によりトップへ蘇生された記事のリスト。
- **リジェクト履歴（審査エラー監視）**: AI審査で弾かれた件数や、エラー（画像なし、薄いコンテンツなど）の理由別グラフ。
- **手動リライト（蘇生）ボタン**: 管理画面から特定の記事を選び、「この記事の感情モードを変えて書き直して」とワンクリックで AI に再評価＆再投稿を命令できる強制上書き機能。

---

## 5. 実装へのロードマップ（今後の予定）

1. `nexus_revive.py` など、単独で稼働するスタンドアロンスクリプトの作成。
2. WP-CLI を用いた「特定記事の公開日（post_date）やカテゴリ、タグの一括更新」ロジックの検証・テスト。
3. DMM/FANZA/DLsite/DigiKet API からの「特定のproduct_idのピンポイント価格情報（定価と現在価格の比較）」の定期取得・差分検知機能の開発。
4. セール終了によるタグ外しの処理（クリーンアップ処理の自動化）の実装。
5. （オプション）Pythonスクリプトとは別に簡単なフロントエンドを用意してダッシュボードUIを構築。
